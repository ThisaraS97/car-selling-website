How to Use EmailJS with Your Website

Step 1: Create an EmailJS Account

Visit EmailJS.com and create a free account. The free tier provides 200 emails per month.

Step 2: Add an Email Service

  1. In your EmailJS dashboard, go to "Email Services" and click "Add New Service"
  2. Select your email provider (Gmail, Outlook, etc.)
  3. Follow the instructions to connect your email account
  4. Give your service a name (e.g., "gmail-service") and save the service
  5. Note down the Service ID that appears

Step 3: Create an Email Template

  1. Go to "Email Templates" in your dashboard and click "Create New Template"
  2. Design your email template with a subject line, content, and variables
  3. Use variables like {{name}}, {{email}}, {{subject}}, and {{message}} - these should match the form field names
  4. Save the template and note down the Template ID

Example template content:

Subject: New Inquiry: {{subject}}

From: {{name}} ({{email}})

Message:
{{message}}

Step 4: Add EmailJS to your Website

Add the EmailJS SDK to your website by adding this script before the closing </body> tag in your HTML:

<script src="https://cdn.jsdelivr.net/npm/@emailjs/browser@3/dist/email.min.js"></script>
<script src="assets/js/contact.js"></script>

Step 5: Update Your Form

Modify your HTML form to have an ID and update the JavaScript file:

<form id="contact-form">
    <!-- Your existing form fields -->
</form>

Step 6: Update the contact.js File

Open the assets/js/contact.js file and update it with your specific information:

  1. Replace YOUR_USER_ID with your EmailJS User ID (found in Account > API Keys)
  2. Replace YOUR_SERVICE_ID with the Email Service ID you created
  3. Replace YOUR_TEMPLATE_ID with the Email Template ID you created

Step 7: Test Your Form

Deploy your updated website and test the contact form to make sure emails are being sent correctly.

Additional Notes

  • The free tier provides 200 emails/month which should be sufficient for testing
  • EmailJS has security measures to prevent abuse, so your form should be secure
  • For more advanced features, you can explore the EmailJS documentation
